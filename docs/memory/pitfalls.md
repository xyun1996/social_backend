# Pitfalls

## Known Risks to Avoid

- Letting tasks redefine scope without first updating `docs/plans/current.md`
- Writing durable decisions only inside chat transcripts
- Mixing business rules into `pkg/`
- Coupling service logic directly to transport-specific concerns
- Treating backlog ideas as committed roadmap items
- Leaving protocol changes undocumented

## Maintenance Reminder

When the same confusion appears twice, promote the answer into `glossary`, `constraints`, `conventions`, ADR, or a runbook.

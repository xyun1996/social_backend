# Social Backend V1.0

- Release date: `2026-03-14`
- Status: `released`

## Summary

`v1.0` delivers a reusable Go social backend baseline for medium-light multiplayer games with a verified local durable runtime and contract-first service boundaries.

## Included Capabilities

- Identity login, refresh, introspection, and player-scoped session handling
- Friend, block, and pending relationship reads
- Shared invite lifecycle with cancel and expiry
- Chat conversations, send, ack, replay, unread summaries, and resource-backed channels
- Guild governance, announcements, logs, progression baseline, and first activity templates
- Party management, queue lifecycle, match handoff, assignment, and resolution cleanup
- Ops reads for player overview, party, guild, worker queue, and durable status
- Durable local MySQL and Redis verification flows

## Verification

- `go test ./...`
- `make check-dev`
- `make test-local-durable`

## Known Gaps

- Guild growth and activities are intentionally lightweight and will need deeper design in `v2`
- Chat moderation and advanced world/system/custom channel governance are deferred
- Social recommendations and richer relationship reads are deferred
- Worker retry and matchmaker lifecycle depth remain intentionally limited
- Production deployment automation is still outside the shipped `v1` scope

## Upgrade / Usage Notes

- Redis local durable tests expect a running Redis instance on the configured local endpoint
- Windows Redis may log `maint_notifications` fallback warnings; this does not block the verified `v1` flows
- Future scope expansion should start from backlog and new version plans, not by widening `v1`
